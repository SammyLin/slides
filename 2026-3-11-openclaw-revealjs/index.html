開始調教龍蝦

Telegram + 語音 = 通勤時間的超能力

然後我遇到了 OpenClaw。 重點是：它接 Telegram。我在通勤的時候，用語音就能指揮 Agent。 語音比打字快三倍，這是 Stanford 的研究。 只有在捷運上或辦公室不方便說話的時候才打字。 【目標時間：60秒】

OpenClaw 不是裝了就會動

用 Markdown 定義一切
個性、背景、規則、記憶
像養小孩一樣
幫牠申請 Gmail、GitHub、Cloudflare
給身分、給工具、給權限
Agent 不是裝了就會動。你要告訴它：你是誰、我是誰、怎麼做事。 SOUL.md 定義個性，USER.md 定義我的背景，AGENTS.md 定義工作規則。 我還幫牠申請了 Gmail、GitHub、Cloudflare，綁信用卡。 就像養小孩 — 給身分、給工具、給權限、給錢。 【目標時間：60秒】

解決「教不會」
通用能力共享，專業知識隔離
痛點一是教不會。解法是：養一群，但共享技能。 四隻 Agent 各司其職，上下文不會錯亂。 通用 Skill 共享一個 Pool，新 Agent 只要教專業知識。 【目標時間：60秒】

成長系統：不會忘、不用催

HEARTBEAT.md
每小時自動醒來
- 檢查信件，有急件通知我
- 看行事曆，開會前提醒
- 追蹤市場，大波動就說
# 我不用問，它會主動來找我
MEMORY.md
每次醒來先讀這個檔案
- 長期記憶寫在 MEMORY.md
- 每日日記寫在 memory/
- 同步到 Obsidian
# 不靠向量 DB，Markdown 就夠
AGENTS.md
做任何事之前，先建 Linear 卡
- 開始做 → 改 In Progress
- 做完 → 記錄成果，改 Done
- 定期回顧，自己反思改進
# 不靠記憶，靠制度

解決「會忘記」：給它心跳、給它記憶、給它流程

痛點二是會忘記。解法是成長系統，三個機制。 Heartbeat：每小時自動醒來巡檢，有事通知我，我不用催。 記憶：MEMORY.md 長期記憶加每日日記，同步到 Obsidian，不需要向量資料庫。 PDCA：做任何事之前先建 Linear 卡，不靠記憶靠流程。 我念了它三次才學會。 【目標時間：75秒】

龍蝦養肥了，要做健康檢查

解決「花太多」：監控 + 模型混搭

github.com/SammyLin/openclaw-exporter

痛點三是花太多。解法是監控加模型混搭。 我是 DevOps 出身，自己寫了 Prometheus Exporter 接 Grafana。 日常對話用 MiniMax 便宜模型，一個月 10 美金。 關鍵任務用 Claude Max 訂閱制 — $100/月。 千萬不要全部用最貴的 API 按 token 算，會被嚇死。 【目標時間：60秒】
